ブランチを使いこなす

作業が錯綜する場合、つまり「これを作ってる途中だけど、やっぱりアレを先に作って」とか言われそうな場合なんかは、特にブランチを使うといいらしいぞ。

ブランチの基本

git checkout -b hoge

今のブランチ（たとえばmaster）を基に新たなブランチ「hoge」を作りつつ、作業スペースを移す。

git add .; git commit -m "…"

hogeブランチに、作業のコミットをしていく…。

git checkout master
git merge hoge

masterブランチに、hogeブランチをマージする。
